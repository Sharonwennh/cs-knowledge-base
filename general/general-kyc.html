🔐 通用-Sumsub KYC 处理指南

最后更新：2026-06-23

适用范围：本文档适用于所有通过 Sumsub 进行 KYC 验证的项目组。处理 KYC 工单时，请务必先查阅本文档，再查阅通用 KYC 规则。

📝 Notes / 注意事项

处理 KYC 前必读以下 7 条规则：
  1. 以下 7 个标签无需操作，提交任务给 T2 并在描述中包含 Tag：
    BLACKLIST / BLOCKLIST / DUPLICATE / FORGERY - rejected / WRONG_USER_REGION / Unsatisfactory age / Images limit reached
  2. 如果辅助显示表格中未列出的标签，提交任务给 T2 并在描述中包含该标签和后台截图。
  3. 表格中的 Key Points for Reply 必须由 T1 编辑后才能使用，绝不能直接发送给玩家。
    部分条目标记为 "Submit a task to the Tier 2 Team" —— 在任何情况下，未经审核不得将此处内容直接发送给玩家。
  4. 如果玩家重置后验证失败 3 次、4 次、5 次或更多次，拒绝记录和尝试次数可在时间线查看。此类情况请提交任务给 T2，描述中包含失败次数和后台截图。
  5. 如果 Reset 和 Approve 按钮同时出现，提交任务给 T2，不要在后台随机点击任一按钮。让 T2 了解此情况并在描述中包含后台截图。
  6. 如果操作按钮显示与表格说明不一致，提交任务给 T2 以防系统潜在错误，并包含后台截图。
  7. 表格中某些标签格式与 Auxiliary 不同。T1 可以逐个单词搜索以避免遗漏。例如：
    Auxiliary 格式：SELFIE_MISMATCH
    Sumsub 格式：invalid ID Spam（此格式对 T1 不可见）

🖥️ KYC Review Panel 字段说明

字段说明
Operations 操作区域。T1 可点击此处按钮更改玩家的 KYC 状态。
Status KYC 状态。显示当前状态，如：Approved（已通过）、Rejected（已拒绝）、Init（初始提交）、Retry（重新提交）。
Level KYC 等级。指示玩家当前的验证层级。
Tags 问题标签。识别 KYC 审核中发现的具体问题。T1 必须使用这些标签在 "Key points for reply Sumsub Applicant" 中定位对应的回复模板。
Note 内部备注。如果 KYC 已被最终拒绝，处理方法（如 1/2/3）将显示在此处。详情参考 Point 4。
Timeline 审核时间线。以日志格式显示 KYC 申请的完整审核历史。
● Red = 失败验证（具体拒绝原因通常出现在条目最后一行）
● Green = 成功验证
Timestamp = 每个条目的时间对应特定操作状态被记录的时刻
Moderation Comment 给申请人的消息。对玩家可见的备注，可能包含拒绝原因。客服需要参考此内容，但仍需匹配到 "Key points for reply Sumsub Applicant" 的正式回复模板。
Action 关键操作日志。记录玩家完成的具体验证步骤，如 Face Authentication 或 Payment Verification。
Status Check：如果验证被拒绝或失败，T1 可在此处找到记录的状态。
● Red = 失败验证尝试
● Green = 成功验证
Timestamp = 每个日志条目显示的时间是验证状态被记录的精确时刻
Created At 提交时间。玩家首次提交 KYC 申请的确切日期和时间。

📍 Point 1: Determining Available Actions

如果 Operations 下没有按钮：

  1. 检查 KYC 状态。如果显示 Init (Initial Submission) 或 Retry，玩家仍可自行更新信息。我方无需操作，也无法从后台操作。
    你可以根据玩家当前的 KYC Level 和 Overview 中玩家的 KYC 验证状态，引导玩家继续完成验证。
  2. 如果玩家声称无法更新信息，请提供录屏并提交任务给 KYC T2。

如果状态是 Rejected：

通常 Reset 或 Approve 按钮会出现在 Operations 下。

  1. Approve：点击此按钮通过 KYC。然后通知玩家。
    ⚠️ 重要：此按钮通常只在 Reason 列指定 AML 且 Tags 列显示 COMPROMISED_PERSONS、ADVERSE_MEDIA 或 PEP 时出现。
    如果 Tags 列显示任何其他不属于 Approve 的标签，不要点击 Approve。而是提交问题给 KYC T2 团队，包括描述和截图。
  2. Reset：点击重置。然后在 "Key points for reply Sumsub Applicant" 中搜索对应标签的回复。
    💡 提示：你可能在 Moderation Comment 列找到通知，但你仍必须将其匹配到 "Key points for reply Sumsub Applicant" 中的回复。

📍 Point 2: Tags Requiring T2 Escalation

立即提交任务给 KYC T2 如果遇到以下 7 个标签：

  1. BLACKLIST
  2. BLOCKLIST
  3. DUPLICATE (multiple accounts issue)
  4. FORGERY - rejected
  5. WRONG_USER_REGION
  6. Unsatisfactory age
  7. Images limit reached
注意：如果 FORGERY 状态是 Init，玩家仍可更新信息，不要提交给 KYC T2。
注意：如果在回复文档中找不到匹配标签且带有 reset 按钮，不要点击 reset，提交任务给 KYC T2。

📍 Point 3: Special Cases for T2 Escalation

  • 如果 Reset 和 Approve 按钮同时出现在 Operations 下，提交任务给 KYC T2。不要在描述中随意操作，包含后台截图。
  • 如果 Operations 下的按钮标签与表格说明不一致，提交任务给 KYC T2 以防系统潜在错误，并包含后台截图。

📍 Point 4: KYC Rejection Process: Note Column 1 & 2 & 3

场景：当玩家的 KYC 被拒绝且 NOTE 指定 1/2 或 3 时，表示 KYC 已被双重检查且无法通过。按以下流程处理。
如果 Note 列有中文文字，请提交 KYC T2 任务并复制此备注请求进一步检查，同时包含截图。

Note 1 — Player HAS Account Balance（玩家有账户余额）

Action：T2 将退还余额并冻结游戏账户。（通常可在退款工作表中找到退款，如果找不到，T2 将在 1 个工作日内录入。）

如果 T2 尚未冻结此玩家，T1 需要先冻结此玩家。

T1 Reply（标准话术）：

Dear Player, We are writing to inform you that, following a review, your account has been permanently suspended due to a failure to pass our mandatory KYC verification. Our team will now process a refund of your remaining account balance. If your total purchase amount is less than the balance to be refunded, the outstanding sum will be issued to you via a direct transfer. This decision is final. Should you have any questions, please contact our support team.

对于 Outstanding Sum（未结清金额）：

  • 如果检查退款工作表发现 T2 填写的 Player Account Balance > Refund amount，意味着有未结清金额，T2 需要跟进处理。
  • 此时，在回复玩家上述绿色标准话术后，需要：
    1. 写下工单备注：
      • a. @Jane 在工单备注中
      • b. 清楚说明玩家 Played SC 余额（例如 80.09 played sc）和实际退款金额（例如 21.99 dollars）导致未结清余额
      • c. 玩家 KYC 被拒绝，风控团队要求退还账户余额（Note 1）
    2. 转派工单给 Jane 并添加 need reply 标签
  • 然后 Jane 将接管此工单直到玩家完全获得未结清退款。
  • 如果实际退款 > 玩家余额（意味着退款成功且完成，如果玩家无其他问题，可以封禁玩家并解决工单）或实际退款待处理（如果玩家询问退款进度，请耐心等待），这是正常的。

Note 2 — Player HAS NO Account Balance（玩家无账户余额）

Action：T2 仅冻结游戏账户。

如果 T2 尚未冻结此玩家，T1 需要先冻结此玩家。

T1 Reply（标准话术）：

Dear Player, We are writing to inform you that, following a review, your account has been permanently suspended due to a failure to pass our mandatory KYC verification. This decision is final. Should you have any questions, please contact our support team.

Note 3 — Refund All Purchase（退还所有购买）

Action：T2 将退还所有购买并冻结游戏账户。（通常可在退款工作表中找到退款，如果找不到，T2 将在 1 个工作日内录入。）

如果 T2 尚未冻结此玩家，T1 需要先冻结此玩家。

T1 Reply（标准话术）：同 Note 1（包含退款说明）。

📍 Point 5: Tag Each Ticket Correctly Based on Actual Operation

Tag使用场景
SA-Approve 当你点击 "Approve" 通过玩家的 KYC（通过 Sumsub）。
SA-Reset 当你通过 Sumsub 重置玩家的 KYC 流程。
SA-T2-Other 当你无法处理 KYC 工单，且 Sumsub applicant 标签不属于以下 7 种场景（BLACKLIST / BLOCKLIST / DUPLICATE / FORGERY / WUR / Unsat-Age / Image-limit）。
SA-T2-BLACKLIST 由于 Sumsub 后台 BLACKLIST 标签无法处理 KYC 工单，需提交给 Tier 2。
SA-T2-BLOCKLIST 由于 Sumsub 后台 BLOCKLIST 标签无法处理 KYC 工单，需提交给 Tier 2。
SA-T2-DUP 由于 Sumsub 后台 Duplicate 标签无法处理 KYC 工单，需提交给 Tier 2。
SA-T2-FORGERY 由于 Sumsub 后台 FORGERY 标签无法处理 KYC 工单，需提交给 Tier 2。
SA-T2-WUR 由于 Sumsub 后台 WRONG_USER_REGION 标签无法处理 KYC 工单，需提交给 Tier 2。
SA-T2-Unsat-Age 由于 Sumsub 后台 Unsatisfactory age 标签无法处理 KYC 工单，需提交给 Tier 2。
SA-T2-Image-limit 由于 Sumsub 后台 Images limit reached 标签无法处理 KYC 工单，需提交给 Tier 2。
SA-IR 不需要 Sumsub 按钮时，只需告诉玩家继续 KYC。或玩家 KYC 被拒需要冻结。
